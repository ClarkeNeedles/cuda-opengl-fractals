# Standalone CUDA-OpenGL Mandelbrot Benchmark Suite

## High-Level Overview
This project is a high-performance standalone Mandelbrot fractal renderer designed using CUDA and OpenGL. Its primary purpose is to serve as a benchmark suite, allowing for the documentation and comparison of performance differences between specialized CPU vector units (e.g., SIMD) and raw GPU core computation. The application maximizes performance through CUDA-OpenGL Interoperability, leveraging Pixel Buffer Objects (PBOs) to write pixel color data directly to the GPU's texture memory, thereby bypassing costly CPU-GPU data transfers.

## System Architecture

The renderer is built as a completely standalone application, ensuring no messy configuration dependencies from other projects. It operates on a continuous frame loop, targeting 60+ FPS, where the complex plane is entirely redrawn on the GPU during user interaction.

### Graphics Pipeline & Interoperability
- **CUDA-OpenGL Interoperability**: The core of the rendering pipeline relies on Pixel Buffer Objects (PBOs) to establish a zero-copy data path. An OpenGL PBO is registered with CUDA using `cudaGraphicsGLRegisterBuffer`, allowing CUDA kernels to directly access and write into OpenGL's texture memory. This eliminates the need to transfer pixel data back to the CPU.
- **Thread Mapping**: Each individual CUDA thread is responsible for calculating the color of exactly one pixel on the screen, enabling massive parallelism.

### CUDA Hardware Foundations
- **Block Execution**: CUDA thread blocks execute entirely on a single Streaming Multiprocessor (SM). An SM can host multiple concurrent blocks, provided sufficient hardware resources (registers, shared memory) are available.
- **Grid Scaling**: The application utilizes a grid of thousands of blocks to fully saturate the GPU's processing capabilities.
- **Warps**: Blocks are subdivided into warps of 32 parallel threads. SMs execute a single instruction for an entire warp simultaneously (SIMT execution).
- **Optimized Configurations**: Block dimensions are chosen to be multiples of 32 (e.g., 16x16 threads per block, forming 8 warps) to maximize hardware utilization and avoid wasting capacity.
- **Coordinate Mapping**: Threads determine their absolute screen position using `blockDim`, `blockIdx`, and `threadIdx` to map their internal block coordinates to global pixel coordinates (`Position_x = blockIdx.x * blockDim.x + threadIdx.x`).

### Mandelbrot Algorithm Implementation
- **Core Equation**: The Mandelbrot set is generated by iterating the sequence `Z_n+1 = Z_n^2 + C`, where `Z₀` starts at 0 and `C` is the complex constant representing the pixel's location.
- **Escape Condition**: The sequence escapes to infinity if `|Z| > 2`. For performance, this is computed as `Z_r^2 + Z_i^2 > 4.0f`.
- **Warp Divergence**: Acknowledges that threads within a warp may take different instruction paths (e.g., some escape early, others run to `max_iters`). This "warp divergence" means all threads in the warp must wait for the longest-running thread, leading to potential hardware underutilization.

### Coloring Implementation
- **Smooth Coloring**: To eliminate color banding, a renormalization formula is applied: `v = iteration - (ln(ln(|Z|)))/ln(2)`.
- **Palette Lookup**: The smoothed value `v` is mapped to a color using a cosine-based function: `color(v) = A + B * cos(2π * (C * v + D))`, utilizing 3-component vectors for bias, amplitude, frequency, and phase. Inside the set (not escaped), pixels default to solid black.

## Tech Stack and Dependencies

-   **CUDA**: Parallel computing platform and programming model for NVIDIA GPUs.
-   **OpenGL**: Cross-platform API for rendering 2D and 3D vector graphics.
-   **GLFW** (or **SDL2**): Library for creating windows, contexts, and handling input.
-   **GLEW** (or **GLAD**): OpenGL Extension Wrangler Library for managing OpenGL extensions.
-   **CMake**: Build system for managing the compilation process.
-   **C++ Compiler**: Compatible with CUDA (e.g., Visual Studio with CUDA build customizations).

## Features

### High-Performance Rendering
-   **CUDA-Accelerated Mandelbrot**: Leverages NVIDIA GPUs for highly parallel fractal generation.
-   **Continuous Frame Loop**: Achieves a target of 60+ frames per second, ensuring fluid animation.
-   **Zero-Copy GPU Data Transfer**: Utilizes CUDA-OpenGL Pixel Buffer Objects (PBOs) for direct GPU memory access, eliminating CPU-GPU data transfer bottlenecks.

### Interactive Exploration
-   **Smooth Zoom and Pan**: Enables seamless exploration of the Mandelbrot set through mouse drag and scroll events.
-   **Cursor-Locked Interaction**: Applies scale adjustments during pans to ensure the fractal remains perfectly locked to the cursor position.

### Precision Handling
-   **Unified Kernel**: Features a template kernel design capable of supporting both `float` (single) and `double` (double) precision mathematics.
-   **Dynamic Precision Hot-Swapping**: Automatically switches between the faster `float` precision path and the deeper `double` precision path once a certain zoom depth threshold (e.g., `1e-7`) is reached, balancing performance with detail.

### Visual Fidelity
-   **Logarithmic Smooth Coloring**: Employs a renormalization formula to eliminate color banding, resulting in visually smooth gradients.
-   **Cosine-Based Color Palette**: Utilizes a flexible cosine function for vibrant and customizable color schemes, producing classic electric blue/purple gradients.
